134 /*--------------------------------------------------------------------------
135  * get/set_cal_xition_date: these routines set the transition date for a Standard
136  * calendar, which is the date that the Gregorian calendar was first used.
137  * Before that, it is assumed that the Julian calendar was used.
138  *
139  * Historically, this transition date varies by country and region. The
140  * variation can be extreme, and over the centuries country boundaries have
141  * changed, so this should be considered only the grossest approximation. For
142  * that matter, for several countries, different districts/regions converted
143  * at different times anyway, so actually doing a good job at this task is
144  * far beyond this library's capability. Nevertheless, this does give some
145  * basic simplified capabilities in this regard. And you can always set
146  * the routines to use an arbitrary transition date of your own calculation.
147  *
148  * How to use these routines:
149  *
150  * If you know the transition date you want to impose, simply call
151  * set_cal_xition_date with the specified transition date. The date
152  * specified is the FIRST date that the new (Gregorian) calendar was
153  * used. For exaple, in Italy, the Gregorian calendar was first used
154  * on 15 October 1582.
155  *
156  * If you don't know what transition date to use, there is a brief
157  * database with some APPROXIMATE dates of transition that can be accessed
158  * by calling get_cal_xition_date with a two-letter country code, corresponding
159  * to the internet suffix for the country. (As a special case, "US" is used
160  * for the United States of America.) If the routine returns 0, then the
161  * country code is recognized and the approximate transition date is returned
162  * in the passed parameters year, month, day. These should then be given to
163  * routine set_cal_xition_date to make that calendar use the specified
164  * transition date. If the get_cal_xition_date routine does not return 0,
165  * then there is no information on that country in the database.
166  *
167  * routine set_cal_xition_date returns 0 on success, and something other than
168  * 0 on an error. Errors include trying to set the transition date to an
169  * invalid date, or trying to set the transition date for any calendar
170  * other than the "Standard" calendar.
171  *
172  * The following country/region codes are recognized: AK (Alaska) 1867/10/18;
173  * AL (Albania) 1912/12/1; AT (Austria) 1583/10/16; BE (Belgium) 1582/12/25;
174  * BG (Bulgaria) 1916/4/1; CN (China) 1929/1/1; CZ (Czechoslovakia) 1584/1/17;
175  * DK (Denmark) 1700/3/1; NO (Norway) 1700/3/1; EG (Egypt) 1875/1/1;
176  * EE (Estonia) 1918/1/1; FI (Finland) 1753/3/1; FR (France) 1582/12/20;
177  * DE (Germany, note different states actually used different dates between
178  * 1583 and 1700!) 1583/11/22; UK (Great Britain and Dominions) 1752/9/14;
179  * GR (Greece) 1924/3/23; HU (Hungary) 1587/11/1; IT (Italy) 1582/10/15;
180  * JP (Japan) 1918/1/1; LV (Latvia) 1915/1/1; LT (Lithuania) 1915/1/1;
181  * LU (Luxemburg) 1582/12/15; NL (Netherlands, note Catholic regions
182  * transitioned in various dates of 1582/83, while Protestant regions
183  * transitioned in various dates of 1700/01) 1582/10/15; NO (Norway) 1700/3/1;
184  * PL (Poland) 1582/10/15; PT (Portugal) 1582/10/15; RO (Romania) 1919/4/14;
185  * ES (Spain) 1582/10/15; SE (Sweden) 1753/3/1; CH (Switzerland, note,
186  * varied bewteen 1584 and 1701 by Canton) 1584/1/22; TR (Turkey)
187  * 1927/1/1; YU (Yugoslavia) 1919/1/1; UK (Great Britain and Dominions) 1752/9/14;
188  * US (United States) 1752/9/14; SU (former Soviet Union) 1918/2/1;
189  * RU (Russia) 1918/2/1.
190  */
191 int ccs_set_xition_date( calcalcs_cal *calendar, int year, int month, int day );
